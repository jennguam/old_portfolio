Frontier Airlines


Overview


This project blossomed out of the prompt -- "Make something better." I chose to redesign Frontier’s Check In process because this was the last time I remembered I was truly frustrated at the design of an interface. Upon research, I discovered that Frontier does not have a responsive mobile web version of their website nor do they have a mobile application. For the purposes of this redesign, I chose to tackle the redesign in the scope of mobile web.


Mobile web is not always considered the most important medium for a product, and it often gets overlooked in favor of desktop or native apps. However, in this case, I would argue that mobile web is very important, for a few reasons:

  • Travelers will largely not bother to download an app to check-in for a flight, unless they travel almost entirely on a single airline.


  • Travelers will often be without laptops or short on time, so they will use a mobile device for some significant portion of check-ins.


  • Checking in to a flight can and should be a quick and easy process.


Challenge


When I approached this project, my challenge was to design a flight check-in process and make it understandable, simple, and as far away as being frustrating. I also only had a couple days to do this with no real brief.

Approach


Define the Problem

Before I do anything, I always try to define the core problem that makes a product or process frustrating. I do this by approaching the problem through the user's eyes. This helps to bring out user pain points.


First, I analyzed the existing flow of how people check into their flights.



There were a number of things that I found to be problematic in the current flow.

1. The reminder email brings the user to a redundant page that outlines the check in options that are available. It adds an extra click. The page has yet another “Check In” button. Wasn’t clicking the “Check In” button from the email already signify that the user wanted to check in online?

2. When you click "Check In" a 2nd time, it directed the user to a magnified unresponsive homepage. I've included what it looks like below. My first reaction was confusion. I didn't know where I was directed to.

Only after I scrolled down, I realized it directed me to an anchored tab where I then would have to fill out the form and have to keep dragging to navigate anywhere. This poses a huge problem, because it will take the user at least 3 taps and 5 drags just to access the reservation.




Second, I took out the redundancies and streamlined the process.




Sketch


I typically do some rough sketches of wireframes to get some ideas on paper.




High Fidelity Mockup


This is my 3rd iteration of the homepage. I started out with what I had in the sketch. The navigation buttons were long rectangular buttons. However, I changed that to size of the buttons to help combat the "fat finger" error. I also changed the background from a more nature theme to one that I think elicits the majesty of flying. I liked this picture particularly because of the subtle, cooler colors that won't distract the user.






After clicking "Check In" from their email, it should direct the user to this page. I've included all the flight information presented in a way that's easy to read and digest. The plane departure and arrival icons can signal to the user where to quickly see when they will be departing or arriving. In the first iteration, I had previously placed the boarding time on the bottom. However, when I really thought about what was necessary, I thought it was more appropriate to have flight status there instead. The Flight Status will be in green, yellow, or red to easily communicate how delayed the flight is.




Complete Check In- I used a stepper control here, because its great for making small adjustments without having to waste any additional clicks. The vast number of travellers typically check around 0-2 bags, so a stepper would make sense. Interacting with drop down menus often requires more effeort than necessary. First, tap the control, scroll, find and select the option, and then exit. The stepper is simple and the function is clear.

With the seating, I wanted to do something different than what currently exists in airline check in processes. I personally hate having to remember if I got the window seat or if I'm stuck in a middle seat. This saves the traveller a click into the seat map, and helps make the check in process faster and easier.





Loading Animation- I started to envision how the flow would be if a user didn't come from a reminder email. The look up process for a reservation typically takes longer, so I wanted to do something interactive and fun while the user waited. It also helps give the user visual feedback that it is doing something in the background. So, I thought this would be the perfect time to pick up Framer! I'm happy to say this is my first animation on Framer and I'm looking forward to doing more.
